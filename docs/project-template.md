# Project Presentation Spec

Every project shown on the personal website must follow this structure.

## Required fields
1. Project Title
2. One-line Summary
3. Context
4. What I Built
5. How It Works
6. Why It Matters
7. Links
8. Optional Image

## Presentation rules
- Write for technical recruiters, hiring managers, and thoughtful peers.
- Emphasize both technical work and real-world impact.
- Do not present projects like generic school assignments.
- Prioritize clarity over buzzwords.
- Keep tone calm, thoughtful, and concrete.
- External links may point to GitHub, demo, article, or write-up.
